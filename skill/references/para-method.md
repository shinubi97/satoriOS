# PARA 方法参考

## 概述

PARA 是 Tiago Forte 在《Building a Second Brain》中提出的知识组织方法。PARA 代表四个类别：Projects、Areas、Resources、Archives。

## 四大类别详解

### 1. Projects (项目)

**定义**: 有明确目标和截止日期的任务。

**特点**:
- 有具体的完成标准
- 有时间限制
- 需要多个步骤完成

**示例**:
- 完成一篇文章
- 学习一门新技能
- 开发一个功能
- 组织一次活动

**目录位置**: `10_项目/`

**笔记结构**:
```markdown
# 项目名称

## 项目目标
- 目标 1
- 目标 2

## 时间线
- 开始: YYYY-MM-DD
- 截止: YYYY-MM-DD

## 关键里程碑
- [ ] 里程碑 1
- [ ] 里程碑 2

## 进展记录
...

## 复盘总结
...
```

### 2. Areas (领域)

**定义**: 持续关注的责任领域，没有明确的完成状态。

**特点**:
- 持续性责任
- 需要长期维护
- 有标准而非目标

**示例**:
- 健康
- 财务
- 职业
- 家庭
- 学习

**目录位置**: `20_领域/`

**笔记结构**:
```markdown
# 领域名称

## 核心关注点
- 关注点 1
- 关注点 2

## 标准
- 标准 1
- 标准 2

## 当前状态
...

## 相关项目
- [[项目 1]]
- [[项目 2]]
```

### 3. Resources (资源)

**定义**: 感兴趣的主题、参考资料和知识资产。

**特点**:
- 主题驱动
- 可复用
- 知识积累

**示例**:
- 机器学习笔记
- 编程语言教程
- 设计原则
- 工具使用指南

**目录位置**: `30_研究/` + `40_资源/`

**笔记结构**:
```markdown
# 资源主题

## 核心概念
- 概念 1
- 概念 2

## 参考链接
- [[相关笔记 1]]
- [[相关笔记 2]]

## 应用场景
...
```

### 4. Archives (归档)

**定义**: 完成的项目、不再活跃的资料和参考资料。

**特点**:
- 保留历史
- 可检索
- 低访问频率

**目录位置**: `50_归档/`

**归档策略**:
- 按月份归档: `50_归档/YYYY-MM/`
- 保留 frontmatter 信息
- 维护反向链接

## PARA 交互流程

```
收件箱 (Inbox)
    ↓
    判断类型
    ├── 项目想法 → /kickoff → Projects
    ├── 研究主题 → /research → Resources
    ├── 参考资料 → 整理 → Resources
    └── 随手记录 → 归档 → Archives

Projects (进行中)
    ↓ 完成后
Archives

Areas (持续关注)
    ← 定期回顾
    ← 关联项目
```

## 核心原则

### 1. 按可操作性组织

笔记的组织依据是"何时需要"，而非"它是什么"。

- Projects: 现在需要
- Areas: 持续需要
- Resources: 可能需要
- Archives: 不再需要

### 2. 最小必要结构

避免过度分类，保持简单：

```
00_收件箱/
10_项目/
    ├── 工作/
    ├── 个人/
    └── 学习/
20_领域/
30_研究/
    ├── 工作/
    ├── 个人/
    └── 学习/
40_资源/
    └── mocs/
50_归档/
    └── YYYY-MM/
Daily/
```

### 3. 链接优先

通过双向链接建立知识网络，而非严格的层级结构。

- 使用 `[[笔记名称]]` 链接相关内容
- 在 MOC 中组织主题笔记
- 利用反向链接发现关联

### 4. 渐进式构建

不要试图一次性建立完美系统：

1. 从收件箱开始
2. 逐步创建项目和领域
3. 自然积累资源
4. 定期归档整理

## 实践指南

### 每日使用

1. **早晨规划** (`/start-my-day`)
   - 查看收件箱
   - 检查进行中项目
   - 规划今日任务

2. **工作中**
   - 记录想法到收件箱
   - 更新项目进展
   - 创建相关笔记

3. **傍晚回顾**
   - 清理收件箱
   - 更新进展
   - 准备次日

### 每周回顾

1. 清理收件箱
2. 更新项目状态
3. 回顾领域关注点
4. 整理资源笔记

### 每月回顾

1. 归档完成项目
2. 更新 MOC
3. 回顾系统结构
4. 调整领域划分

## 常见问题

### Q: 如何区分项目和领域？

**项目**:
- 有明确的"完成"状态
- 有截止日期
- 例: 写完一篇文章

**领域**:
- 持续的责任
- 没有完成状态
- 例: 写作能力

### Q: 资源和研究笔记的区别？

**资源**:
- 面向主题
- 知识积累
- 参考、查阅

**研究笔记**:
- 面向学习过程
- 学习记录
- 渐进构建

### Q: 何时归档？

- 项目完成
- 笔记 3 个月未访问
- 领域不再关注
- 研究主题结束
