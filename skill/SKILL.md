# Obsidian KB - 知识库管理工作流

## 概述

Obsidian KB 是一套基于 PARA 方法论的知识管理工作流，帮助用户高效管理 Obsidian 笔记库。通过结构化的工作流，实现从想法到项目、从研究到知识沉淀的全生命周期管理。

## 核心理念

### PARA 方法论

- **Projects (项目)**: 有明确目标和截止日期的任务
- **Areas (领域)**: 持续关注的责任领域
- **Resources (资源)**: 感兴趣的主题和参考资料
- **Archives (归档)**: 完成的项目和不再活跃的资料

### 核心原则

1. **渐进式构建**: 从想法开始，逐步完善
2. **链接优先**: 通过双向链接建立知识网络
3. **定期回顾**: 通过每日规划和健康检查保持系统活力
4. **精华提取**: 从头脑风暴和研究中提取可行动的知识

## 目录结构

```
vault/
├── 00_收件箱/          # 待处理的想法和笔记

│   └── 学习/

```

## 工作流程

### 1. 每日规划 (/start-my-day)

**触发时机**: 开始一天的工作时

**执行方式**:
```bash
obsidian-kb start-my-day
```

**工作流程**:
1. 扫描收件箱（最多 5 个笔记）
2. 列出进行中项目
3. 提取最近 7 天的待办项
4. 生成今日建议

**输出示例**:
```
📅 2026-03-31 每日规划

## 收件箱 (3)
- 学习 Rust 的想法 [项目想法]
- 周会记录 [归档]
- 新功能设计 [研究]

## 进行中项目 (2)
- [[知识库重构]] - 编程 [进行中]
- [[博客文章]] - 写作 [进行中]

## 今日待办 (5)
- [ ] 完成 README 编写
- [ ] 复习昨天的笔记

## 建议
- 处理 1 个项目想法 (使用 /kickoff)
- 开始 1 个研究主题 (使用 /research)
```

### 2. 项目启动 (/kickoff)

**触发时机**: 收件箱有新的项目想法

**执行方式**:
```bash
obsidian-kb kickoff "项目名称" --area=领域 --timeline=时间线
```

**参数**:
- `项目名称`: 要启动的项目名称（会在收件箱中搜索匹配）
- `--area`: 所属领域（可选，默认使用配置）
- `--timeline`: 预期完成时间（可选，默认 1 个月）
- `--goals`: 项目目标列表（可选）

**工作流程**:
1. 在收件箱中查找匹配的想法
2. 创建项目笔记（使用模板）
3. 归档原想法到 `50_归档/YYYY-MM/`
4. 返回项目笔记路径

**示例**:
```bash
obsidian-kb kickoff "学习Rust" --area=编程 --timeline="2个月" --goals="掌握基础语法,理解所有权机制"
```

### 3. 研究笔记 (/research)

**触发时机**: 开始新的学习或研究主题

**执行方式**:
```bash
obsidian-kb research "主题" --area=领域 --depth=深度
```

**参数**:
- `主题`: 研究主题
- `--area`: 所属领域（可选）
- `--depth`: 研究深度
  - `快速了解`: 快速概览，了解基本概念
  - `深入学习`: 系统学习，掌握核心知识（默认）
  - `精通掌握`: 全面精通，能够实践应用

**深度对应结构**:
- 快速了解: 核心概念 → 快速入门 → 参考资料
- 深入学习: 核心概念 → 详细说明 → 实践笔记 → 学习资源
- 精通掌握: 核心概念 → 深入原理 → 最佳实践 → 进阶主题 → 项目实战

**示例**:
```bash
obsidian-kb research "机器学习入门" --area=编程 --depth="深入学习"
```

### 4. 头脑风暴 (/brainstorm)

**触发时机**: 需要深入思考某个问题或主题

**执行方式**:
```bash
obsidian-kb brainstorm "主题" --project=项目 --area=领域
```

**参数**:
- `主题`: 头脑风暴主题
- `--project`: 关联项目名称（可选）
- `--area`: 所属领域（可选）

**工作流程**:
1. 创建头脑风暴笔记
2. 如果关联项目，存储在项目目录；否则存储在研究目录
3. 支持多轮对话追加内容
4. 对话结束后可提取精华

**交互模式**:
Agent 会创建笔记并返回路径，用户可以继续对话追加内容：
```
用户: 我想讨论一下如何提高学习效率
Agent: ✅ 已创建头脑风暴笔记: 30_研究/学习/如何提高学习效率_头脑风暴_2026-03-31.md

用户: 我觉得番茄工作法很有效
Agent: 已追加到笔记...

用户: 继续讨论间隔重复学习法
Agent: 已追加...

用户: 帮我提取精华
Agent: 
## 精华提取

### 核心结论
- 番茄工作法通过时间分块提高专注度
- 间隔重复利用遗忘曲线强化记忆

### 可行方案
1. 25分钟工作+5分钟休息循环
2. 使用 Anki 进行间隔重复

### 下一步行动
- [ ] 今天尝试番茄工作法
- [ ] 下载 Anki 并创建第一个牌组
```

### 5. 归档 (/archive)

**触发时机**: 项目完成或笔记不再活跃

**执行方式**:
```bash
obsidian-kb archive "笔记路径"
```

**工作流程**:
1. 移动笔记到 `50_归档/YYYY-MM/`
2. 更新 frontmatter 状态
3. 返回归档路径

### 6. 知识地图 (/mocs)

**触发时机**: 查看或维护 MOC

**执行方式**:
```bash
obsidian-kb mocs                    # 列出所有 MOC
obsidian-kb moc "领域"              # 查看特定领域 MOC
obsidian-kb moc-update "领域"       # 更新 MOC
```

**MOC 内容包括**:
- 核心笔记链接
- 相关项目
- 研究笔记
- 头脑风暴记录
- 学习资源
- 待探索主题

## Frontmatter 规范

所有笔记使用统一的 frontmatter 格式：

```yaml
---
id: kb-20260331-160933-dyr8
title: 笔记标题
type: project|research|brainstorm|daily|moc|idea
area: 编程
date: '2026-03-31'
created: 2026-03-31 16:09
updated: 2026-03-31 16:09
tags:
- 标签1
- 标签2
status: 进行中|活跃|已完成|已归档
---
```

### ID 格式

`kb-YYYYMMDD-HHMMSS-XXXX`

- 日期: YYYYMMDD
- 时间: HHMMSS
- 随机后缀: XXXX（4位字母数字）

### 必需字段

- `id`: 唯一标识符
- `title`: 笔记标题
- `type`: 笔记类型
- `date`: 创建日期
- `area`: 所属领域

## 与 Agent 协作

### 启动工作流

当用户请求某个工作流时，Agent 应：

1. **确认参数**: 如果缺少必要参数，询问用户
2. **执行工作流**: 调用对应的 CLI 命令或 Python API
3. **展示结果**: 清晰展示工作流输出
4. **后续建议**: 根据结果提供下一步建议

### 多轮对话

对于头脑风暴等需要多轮对话的工作流：

1. 创建初始笔记
2. 接收用户输入，追加到笔记
3. 用户明确结束或请求提取精华时，进行总结

### 文件操作

Agent 可以使用 Obsidian CLI 进行文件操作：

```bash
# 创建笔记
obsidian create "笔记标题" --folder=文件夹

# 编辑笔记
obsidian edit "笔记路径" --append="追加内容"

# 搜索笔记
obsidian search "搜索词"

# 打开笔记
obsidian open "笔记路径"
```

## 最佳实践

### 每日使用

1. **早晨**: 执行 `/start-my-day` 获取今日规划
2. **处理收件箱**: 将想法转化为项目或研究笔记
3. **记录进展**: 更新项目笔记的进展记录
4. **傍晚**: 回顾完成情况，准备次日

### 项目管理

1. 使用 `/kickoff` 启动新项目
2. 定期更新项目进展
3. 项目完成后使用 `/archive` 归档
4. 进行复盘总结

### 知识积累

1. 使用 `/research` 创建研究笔记
2. 学习过程中填充内容
3. 提取核心知识到独立笔记
4. 更新相关 MOC

### 头脑风暴

1. 明确讨论主题
2. 自由发散想法
3. 归纳整理
4. 提取精华和行动项
5. 关联到相关项目

## 故障排除

### 笔记找不到

- 检查目录结构是否正确
- 确认 frontmatter 格式有效
- 使用 `obsidian search` 搜索

### 工作流执行失败

- 检查 Obsidian CLI 是否安装
- 确认 vault 路径配置正确
- 查看错误日志

### 链接失效

- 运行 `/health-check` 检查链接完整性
- 手动修复或删除失效链接
